# 项目开发计划：扫地机器人智能客服系统

## 一、项目概述

- **项目名称**：RAG-CleanServ
- **类型**：基于 LangChain 1.2+ 构建的混合检索 RAG + 智能 Agent 智能客服系统
- **核心功能**：
  1. **智能问答**：采用向量检索 + BM25 关键词检索 + RRF 融合 + 重排的混合检索方案，依托产品知识库结合大模型生成回答
  2. **报告生成**：根据已购买用户的使用记录，自动生成个性化使用报告
  3. **多级 MD5 去重与缓存**：文件级、文本块级、问题级三层 MD5 校验，规避重复解析、重复向量化、重复 LLM 调用，提升响应速度
- **技术栈**：Python 3.10+、LangChain、DeepSeek LLM API、ChromaDB、BGE 中文 Embedding、BM25、Rerank 重排、Streamlit、高德天气 API、MD5 工具

## 阶段 2：知识库、混合检索 & 多级 MD5 去重模块（2 天）
- [ ] 收集并整理6份产品知识库文档（故障排除、常见对答、维护保养、产品说明、使用技巧、配件信息），存放至 `data/raw/`
- [ ] 实现文档加载、分块（`RecursiveCharacterTextSplitter`，`chunk_size=500`）
- [ ] 实现多级 MD5 去重逻辑：文件级校验、文本块级校验
- [ ] 实现向量库构建与持久化（`ChromaDB` + BGE 中文 Embedding）
- [ ] 接入 BM25 关键词检索，搭配 jieba 实现中文分词检索
- [ ] 开发 RRF 排序融合算法，合并向量检索与 BM25 检索结果
- [ ] 接入 Rerank 重排模型，对召回文档二次打分排序（截取 Top3）
- [ ] 实现 `rag_summary` 函数：
  - 从向量库+BM25检索相关文档（k=3~5）
  - RRF融合排序 → Rerank重排筛选
  - 上下文拼接 → 调用 LLM 生成答案
- [ ] 测试 RAG 问答效果（单独运行脚本，调试检索参数）

## 五、风险与应对

| 风险                   | 应对措施                                                                                       |
| ---------------------- | ---------------------------------------------------------------------------------------------- |
| OpenAI API 调用成本高  | 1. 测试阶段使用 `gpt-3.5-turbo` <br>2. 本地模拟部分响应 <br>3. 设置单日预算上限                |
| 向量检索效果差         | 1. 调试 chunk 重叠参数 <br>2. 尝试替代 embedding（如 `bge-large-zh`）<br>3. 添加关键词前置过滤 |
| 报告生成所需数据不存在 | 先构建模拟数据 `mock_usage_records.json`，后续接入真实数据库                                   |
| Agent 工具调用顺序混乱 | 使用 `handle_parsing_errors=True`，并在 Prompt 中明确工具选择规则                              |
